# Video2MDNotes User Guide (Legacy Scripts)

This guide describes how to use the legacy `video2mdnotes` scripts. These scripts are still functional but have been superseded by the modern Python application.

## Prerequisites

Before running the scripts, ensure you have the following installed:

1.  **Python 3.10+**
2.  **FFmpeg** (Required by yt-dlp and whisper)
    *   `brew install ffmpeg`
3.  **Python Dependencies**:
    *   `pip install faster-whisper rich`
4.  **LLM CLI Tool** (Required for summarization):
    *   Install: `pip install llm`
    *   Setup OpenAI Key: `llm keys set openai`
    *   (Or configure other models as needed)
    *   **Note:** This tool is by Simon Willison. See [llm.datasette.io](https://llm.datasette.io/) for full documentation.
5.  **yt-dlp**:
    *   `brew install yt-dlp`

## Workflow

### Step 1: Download Audio
Use `yt-dlp` to download the audio from your video source into the project root. We specifically want `.wav` format for the transcriber.

**Basic Command:**
```bash
yt-dlp -x --audio-format wav "YOUR_VIDEO_URL"
```

**With Browser Cookies (for Udemy/Coursera/etc):**
```bash
yt-dlp -x --audio-format wav --cookies-from-browser chrome "YOUR_VIDEO_URL"
```

### Step 2: Clean Filenames
Run the cleanup script to normalize filenames (lowercase, no spaces, date prefix). This prevents issues with downstream tools.

**Dry Run (Check what will happen):**
```bash
./clean_dir_filenames.sh --dry-run
```

**Execute Rename:**
```bash
./clean_dir_filenames.sh
```

### Step 3: Transcribe Audio
Run the Python script to transcribe all `.wav` files currently in the project root.

```bash
python3 transcribe_dir.py
```

*   **Input:** Reads `.wav` files from project root.
*   **Output:** Creates Markdown transcripts in `transcripts/`.
*   **Cleanup:** Moves processed `.wav` files to `transcribed_wav/`.

### Step 4: Generate Summaries
Run the batch script to send transcripts to the LLM for summarization.

```bash
./batch_run.sh
```

*   **Input:** Reads `.md` files from `transcripts/`.
*   **Prompt:** Uses `summarize_prompt.txt`.
*   **Output:** Creates final notes in `summaries/`.
*   **Cleanup:** Moves the original `.wav` files from `transcribed_wav/` to `transcribed_wav/archive/`.

---

## Directory Structure Overview

*   **Root**: Place new `.wav` downloads here.
*   **`transcripts/`**: Intermediate raw transcripts (generated by Step 3).
*   **`summaries/`**: Final AI-generated notes (generated by Step 4).
*   **`transcribed_wav/`**: Holding area for audio that has been transcribed but not summarized.
*   **`transcribed_wav/archive/`**: Final resting place for fully processed audio files.
